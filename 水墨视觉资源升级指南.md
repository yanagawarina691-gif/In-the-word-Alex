# 《字里行间》水墨视觉资源升级指南

## 1. 文档目的

本文档用于指导《字里行间》的视觉资源搜集、制作与后续接入。

当前项目中的小球、月亮和特效主要由 CSS 渐变与 Canvas 2D 基础图形生成：

- 小球：CSS 圆形、渐变、高光和阴影
- 月亮：CSS 圆形、径向渐变和光晕
- 墨尘、碎片、冰霜：Canvas `fillRect()` 绘制的小型矩形粒子
- 动画：GSAP 负责位移、缩放、旋转和时间线编排

现阶段的主要问题不是动画库不足，而是视觉资产过于依赖规则几何、简单渐变和程序化方块。升级重点应放在统一的水墨素材体系上。

## 2. 核心视觉设定

### 2.1 视觉命题

小球不是金属球，而是一滴从诗句墨迹中醒来的“墨灵”。

整个世界应像一张安静的宣纸：

- 文字是世界本身
- 墨灵在字句之间行走
- 月亮由“明月”二字的墨迹凝结而成
- 冰霜表现为纸面、墨迹与纤维的状态变化
- 碰撞、跳跃和解锁都通过墨迹的聚散表达

### 2.2 风格关键词

- 宣纸
- 焦墨
- 淡墨
- 飞白
- 渗化
- 留白
- 月华
- 纸面纤维
- 克制
- 安静

### 2.3 应避免的方向

- 金属质感小球
- 高饱和蓝色冰晶
- 卡通星星和闪光
- 常见手游粒子
- 规则圆形光斑
- 纯黑且边缘光滑的墨块
- 写实程度过高的天文月球
- 明显的霓虹、玻璃或塑料质感

## 3. 水墨精灵

### 3.1 角色形态

墨灵应保留圆形轮廓，确保在小尺寸下容易辨认，同时加入以下特征：

- 略微不规则的外轮廓
- 干笔或渗墨形成的边缘
- 暖黑、焦墨和淡墨的内部层次
- 一个极小的留白高光
- 运动时产生轻微变形
- 停止时像一滴仍有生命的墨

当前角色显示尺寸为 15px。升级后建议显示为 18～22px，源素材按照 128×128px 或 256×256px 制作，再缩小显示。

### 3.2 动画资源清单

| 状态 | 建议帧数 | 动作描述 |
| --- | ---: | --- |
| 待机呼吸 | 6帧 | 轮廓缓慢收缩、舒展，内部墨色轻微流动 |
| 向前滚动 | 8帧 | 墨团滚动，轮廓和留白高光随旋转变化 |
| 起跳压缩 | 2帧 | 水平变宽、垂直压缩 |
| 腾空舒展 | 2～4帧 | 向上拉长，边缘出现少量飞白 |
| 下落 | 2～4帧 | 逐渐恢复圆形，底部略微收紧 |
| 落地回弹 | 4帧 | 接触纸面后压扁，再快速恢复 |
| 撞墙变形 | 3～4帧 | 接触侧扁，溅出少量墨点 |
| 冰冻状态 | 4帧 | 墨色停滞、边缘发涩、表面浮现霜纹 |
| 下沉状态 | 4～6帧 | 逐渐被纸面或文字吸收 |

### 3.3 拖尾资源

准备 6～10 张独立拖尾贴图：

- 墨点
- 短墨线
- 干笔断痕
- 淡墨雾
- 细小飞白碎屑

拖尾不需要做成完整逐帧动画。程序可以随机选择贴图，并调整旋转、缩放、透明度和生命周期。

### 3.4 素材规格

- 格式：透明 PNG 或透明 WebP
- 单帧尺寸：128×128px 或 256×256px
- 色彩：建议灰度或低饱和暖黑
- 主体位置：保持在统一中心点
- 边缘：保留真实透明度，不要使用硬抠边
- 输出方式：单帧文件和 Sprite Sheet 各保留一份
- 命名示例：`ink-spirit-idle-01.webp`

## 4. 月亮系统

### 4.1 分层原则

月亮不应使用一张完整黄色圆形图片。建议拆分为：

1. 月体：主体轮廓和月面纹理
2. 月晕：大范围、低透明度的光
3. 边缘散射：靠近月体的柔光
4. 月尘：缓慢漂浮的小颗粒和淡墨云气
5. 撞击层：月亮受力时出现的墨纹、涟漪或碎屑

### 4.2 状态清单

| 状态 | 表现 |
| --- | --- |
| 文字凝聚 | “明月”二字碎成墨片，墨片向中心聚集 |
| 初次出现 | 月体由模糊变清晰，月晕稍晚出现 |
| 静止悬浮 | 极轻微呼吸、明暗和上下漂移 |
| 被撞击 | 月体短暂变形，月晕产生环形波动 |
| 坠落或下沉 | 光晕先拖长，月体随后下降 |
| 场景交接 | 月亮成为下一幕的视觉锚点 |

### 4.3 素材规格

- 月体：512×512px 或 1024×1024px
- 月晕：透明 PNG/WebP，1024×1024px
- 月尘：128×128px 或 256×256px 单张贴图
- 月体边缘：轻微宣纸渗墨感，不要绝对光滑
- 月面纹理：偏水墨和米金，不要过度写实
- 发光：应偏柔和月华，不要形成黄色霓虹灯

### 4.4 搜索关键词

## 5. 墨迹粒子系统

### 5.1 资源清单

| 类型 | 建议数量 | 用途 |
| --- | ---: | --- |
| 圆形墨点 | 8～12张 | 行走、跳跃和普通碰撞 |
| 干笔碎屑 | 6～8张 | 撞墙、文字破碎 |
| 短墨线 | 4～6张 | 快速移动和方向表达 |
| 淡墨雾 | 4张 | 月亮生成、慢动作和下沉 |
| 撞击飞溅 | 4～6张 | 撞击“明月”或软墙 |
| 落地墨圈 | 3～4张 | 落地回弹 |
| 墨水扩散遮罩 | 4～8张 | 字体点亮和纸面渗化 |

### 5.2 素材要求

- 透明背景
- 每张形状明显不同
- 同时包含浓墨、淡墨和飞白
- 建议尺寸为 128×128px
- 黑色区域应保留透明度层次
- 避免外发光、硬边和纯黑剪影

### 5.3 后续接入方式

Canvas 粒子由当前的 `fillRect()` 改为 `drawImage()`：

- 随机选择粒子贴图
- 随机旋转
- 随机缩放
- 随机透明度
- 根据速度拉伸
- 根据粒子寿命逐渐淡出
- 使用 `multiply` 或适合纸面的混合模式

暂时不需要引入 Three.js。Canvas 2D 足以完成这一阶段的升级。

## 6. 冰霜系统

### 6.1 视觉方向

冰霜不应表现为常见的蓝色冰块或雪花，而应表现为“纸与墨被冻结”：

- 宣纸纤维逐渐发白
- 墨灵边缘变得干涩
- 淡灰蓝霜纹从接触点扩散
- 周围出现低透明度冷雾
- 诗字墨色短暂变浅
- 动作变慢并带有轻微滞涩感

### 6.2 资源清单

- 霜纹：4～6张
- 冷雾：4张
- 细小冰屑：3～4张
- 纸面冻结遮罩：1～2张
- 墨灵冻结覆盖纹理：2～4张

## 7. 宣纸背景

### 7.1 资源组成

建议将背景拆为：

1. 基础纸色
2. 无缝宣纸纹理
3. 极淡纸张纤维
4. 少量自然斑点
5. 边缘轻微明暗变化
6. 场景状态叠色

### 7.2 状态版本

- 暖宣纸：正常阅读和月光出现前
- 冷宣纸：慢动作和冻结状态
- 亮宣纸：月光爆发和场景转换

纸张纹理必须克制。正常观看时不应抢眼，但关闭纹理后应能明显感到画面变平。

### 7.3 素材规格

- 推荐尺寸：2048×2048px
- 优先使用无缝纹理
- 可以使用 JPG、WebP 或 AVIF
- 不要带明显折痕、污渍或摄影光影
- 移动端压缩后仍需避免色带和块状噪声

### 7.4 搜索关键词

## 8. 文字与墨的互动

### 8.1 可升级效果

- 经过“光”字：墨色向外轻微晕开
- 触发“霜”字：文字边缘出现纸纤维般的霜纹
- 进入慢动作：与月亮有关的文字产生悬浮和墨气
- 撞击“明月”：文字碎成真实笔触，不再是规则方块
- 月亮形成：墨片聚合为月体
- 解锁下半句：墨水重新渗入文字
- 下沉：角色被“低”字或纸面吸收

### 8.2 可搜集资源

- 墨水扩散遮罩
- 毛笔扫过纹理
- 字体碎裂笔触
- 墨迹吸入序列
- 墨团聚合序列
- 宣纸吸墨录像或参考素材

### 8.3 搜索关键词

## 9. 声音资源

视觉升级后，声音应采用低存在感的纸、墨和空气质感。

### 9.1 资源清单

- 纸面轻擦
- 墨滴落纸
- 毛笔扫过宣纸
- 轻微玉石碰撞
- 月亮形成时的低频空气声
- 冻结时的纤维收紧声
- 文字碎裂时的干纸或细砂声
- 场景转换时的短促留白

### 9.2 应避免

- 金币声
- 卡通跳跃声
- 明显的菜单提示音
- 玻璃碎裂声
- 高音量冰冻音效
- 长混响的影视预告音效

## 10. 资源授权与来源记录

- 文件名

- 下载日期
- 授权协议
- 是否允许商业使用
- 是否要求署名
- 是否允许修改
- 项目中的使用位置

推荐优先选择：

- CC0
- 公有领域
- 明确允许商业使用与修改的授权
- 自制或委托制作的原创资源

## 12. 搜集优先级

### 第一阶段：建立角色质感

1. 水墨精灵待机、滚动、跳跃和落地帧
2. 墨点与干笔粒子
3. 一张可用的暖宣纸纹理

这一阶段完成后，小球与页面应不再像普通网页元素。

### 第二阶段：建立核心演出

1. 月体
2. 月晕
3. 月尘
4. 文字碎裂笔触
5. 月亮凝聚和撞击素材

这一阶段完成后，“撞下明月”应成为第一关的主要记忆点。

### 第三阶段：完善状态反馈

1. 冰霜纹理
2. 冷雾
3. 纸面冻结遮罩
4. 墨水扩散遮罩
5. 下沉与场景交接素材

### 第四阶段：加入声音

在视觉节奏稳定后再匹配声音，避免声音长度反过来限制动画调整。

## 13. 单项资源验收标准

找到资源后，逐项检查：

- 是否符合水墨和宣纸语言
- 缩小到实际尺寸后是否仍能辨认
- 是否有明显白边或抠图痕迹
- 是否带有不需要的背景
- 是否过于卡通、写实或游戏化
- 是否能与黑色诗字共存
- 是否有足够透明度层次
- 是否可以自然地循环或随机组合
- 是否有明确、可追踪的授权
- 是否经过压缩后仍然稳定

## 14. 技术升级边界

第一阶段继续使用：

- DOM
- CSS
- Canvas 2D
- GSAP

优先将简单 CSS 渐变和矩形粒子替换成高质量贴图。

只有出现以下需求时，再考虑 PixiJS 或 WebGL：

- 同屏粒子数量明显增加
- 需要实时位移、噪声或液体 Shader
- 需要大量混合模式和后处理
- Canvas 2D 在目标设备上性能不足
- 需要更复杂的遮罩、滤镜和纹理变形

Three.js 不应作为当前二维关卡的默认升级方案。更复杂的技术不会自动带来更好的视觉品质。

## 15. 最终目标

第一关完成视觉升级后，应达到以下体验：

- 墨灵看起来属于诗句和宣纸，而不是覆盖在页面上的按钮
- 月亮像从文字中生成，而不是突然显示的一张图
- 粒子具有真实笔触差异，不再是程序化方块
- 冰霜改变纸与墨的状态，而不是添加蓝色装饰
- 所有动画虽然克制，但每个动作都有材料感
- 即使关闭复杂特效，角色、纸张和月亮本身仍然成立
